# If-you-are-the-one 设计文档

> 人员匹配与自组队平台——让黑客松、工作坊等现场活动的组队真正高效。

---

## 一、问题陈述

### 现状痛点

在黑客松、工作坊、创意 Jam 等需要**现场组队**的活动中，组队环节长期存在以下痛点：

| 痛点 | 描述 |
|------|------|
| **现场低效** | 几十上百人面面相觑，自我介绍+试探+磨合耗时巨大 |
| **预先组队浅** | 线上组队仅靠几句文字对话，信息维度单一，匹配质量差 |
| **信息不对称** | 不知道谁在找队、谁有什么技能、谁和自己互补 |
| **性格盲区** | 技能匹配≠合作匹配，性格、工作风格、期望差异导致冲突 |
| **社恐壁垒** | 内向参与者难以主动破冰，人才被埋没 |
| **组队后散** | 队伍缺乏凝聚力，活动结束即解散，无后续协作 |

### 目标

构建一个**活动组队匹配平台**，让参与者在活动开始前就能：
1. 充分了解彼此（技能、性格、期望）
2. 发现最互补的潜在队友
3. 自然形成团队，而不是被随机分配

---

## 二、产品定位

**名称**：If-you-are-the-one（非诚勿扰）

**隐喻**：借鉴相亲节目"非诚勿扰"的模式——不是随机配对，而是让参与者在充分展示和了解后，做出有信息量的选择。

**核心理念**：
- 不是"帮你找到最佳队友"的算法黑盒
- 而是"让你看见全局、做出最好选择"的信息平台
- 从 ClawMatch 的**被动展示**进化为**主动匹配+组队行动**

## 三、从 ClawMatch 继承的设计理念

通过分析 ClawMatch（MistrE）项目，以下设计理念直接适用：

### 3.1 可复用的理念

| ClawMatch 理念 | If-you-are-the-one 继承方式 |
|----------------|--------------------------|
| **力导向图可视化** | 保留。人-技能标签二部图是核心展示形式 |
| **标签关系网络** | 保留并增强。标签关联 + BFS 传递发现间接匹配 |
| **Canvas 多空间隔离** | 演化为"活动(Event)"概念——每个活动就是一个独立空间 |
| **Agent 代创建档案** | 保留。活动主办方可批量导入参与者 |
| **Claim 认领机制** | 保留。参与者认领预创建的档案并补充信息 |
| **WebSocket 实时更新** | 保留。现场使用时图谱即时响应 |
| **邀请码体系** | 演化。活动邀请码 = 进入特定活动空间的入口 |

### 3.2 需要突破的局限

| ClawMatch 局限 | If-you-are-the-one 演进 |
|----------------|----------------------|
| 仅展示网络，无主动匹配 | 新增：匹配推荐引擎 |
| 无团队概念 | 新增：队伍(Team) 实体 + 组队流程 |
| 无需求/供给区分 | 新增：每人声明"我能提供"和"我需要"的技能 |
| 无偏好/意向表达 | 新增：兴趣标记(Like)、组队邀请 |
| 无性格维度 | 新增：轻量性格/工作风格标签 |
| 无活动生命周期 | 新增：活动阶段流转（报名→匹配→组队→锁定） |
| 无队伍管理 | 新增：队伍创建、加入、确认、锁定 |

## 四、核心概念模型

### 4.1 实体关系

```
Event (活动)
 ├─ 1:N ── Participant (参与者)
 │          ├─ Profile (档案)
 │          │   ├─ skill_offer: [技能标签]    ← 我能提供
 │          │   ├─ skill_want: [技能标签]     ← 我在找
 │          │   ├─ work_style: [风格标签]     ← 偏好工作方式
 │          │   └─ bio / avatar / 联系方式
 │          ├─ Interests (兴趣标记)
 │          │   └─ liked_participants: [id]
 │          └─ Team Membership
 │              └─ team_id, role, status
 │
 ├─ 1:N ── Team (队伍)
 │          ├─ name, description
 │          ├─ idea: String               ← 队伍想做什么
 │          ├─ skill_slots: [标签+数量]   ← 还缺什么人
 │          ├─ members: [Participant]
 │          ├─ status: recruiting|full|locked
 │          └─ created_by: Participant
 │
 ├─ 1:N ── Tag (标签)
 │          ├─ category: skill|style|domain
 │          └─ relationships: [Tag]
 │
 └─ Lifecycle (生命周期)
            ├─ phase: setup|registration|matching|teaming|locked
            └─ config: max_team_size, deadline, etc.
```

### 4.2 核心标签分类

| 分类 | 示例 | 用途 |
|------|------|------|
| **技能标签** | 前端、后端、UI设计、数据分析 | 匹配互补技能 |
| **领域标签** | AI、Web3、教育、医疗 | 匹配兴趣方向 |
| **风格标签** | 快速迭代、深度研究、视觉驱动 | 匹配工作方式 |
| **角色标签** | 队长、执行者、创意者、协调者 | 匹配团队角色 |

## 五、活动生命周期

```
┌──────────┐     ┌──────────────┐     ┌──────────────┐     ┌──────────────┐     ┌──────────┐
│  Setup   │────▶│ Registration │────▶│   Matching   │────▶│   Teaming    │────▶│  Locked  │
│ 活动配置  │     │   报名入驻    │     │   匹配发现    │     │   自由组队    │     │  锁定开赛 │
└──────────┘     └──────────────┘     └──────────────┘     └──────────────┘     └──────────┘
  主办方创建       参与者填写档案       推荐+可视化探索     创建/加入队伍        组队截止
  设定规则         认领、补充信息       标记兴趣、交流      确认成员            开始活动
```

### 各阶段功能

| 阶段 | 主办方能力 | 参与者能力 |
|------|-----------|-----------|
| **Setup** | 创建活动、设定规则、上传参与者名单 | — |
| **Registration** | 发送邀请码、监控入驻进度 | 创建/认领档案、填写技能标签 |
| **Matching** | 查看全局图谱、推荐引擎调参 | 浏览网络图、查看推荐、标记兴趣 |
| **Teaming** | 监控组队进度、处理争议 | 创建/加入队伍、确认队友 |
| **Locked** | 导出队伍列表、开始活动 | 查看最终队伍、开始协作 |

## 六、匹配引擎设计

### 6.1 匹配维度

匹配评分 = 加权综合以下维度：

| 维度 | 权重 | 计算方式 |
|------|------|----------|
| **技能互补度** | 0.40 | A 的 skill_offer ∩ B 的 skill_want + 反向 |
| **领域契合度** | 0.25 | 共同领域标签（含标签关系传递） |
| **风格相容度** | 0.15 | 工作风格标签的兼容性矩阵 |
| **双向兴趣** | 0.15 | 互相标记了兴趣加分 |
| **队伍需求** | 0.05 | 对方队伍的 skill_slots 与自己的 offer 重合 |

### 6.2 技能互补算法

```
互补度(A, B) = |A.offer ∩ B.want| + |B.offer ∩ A.want|
             ────────────────────────────────────────
                   |A.want| + |B.want|

传递扩展：
  A.offer 中的每个标签，通过标签关系 BFS 扩展为"等效技能集"
  B.want  中的每个标签，同样扩展
  扩展后再计算交集，但权重随 BFS 深度衰减（depth 1: 1.0, depth 2: 0.5, depth 3: 0.25）
```

### 6.3 推荐展示

每个参与者看到的推荐列表：

```
┌─ 推荐理由卡片 ─────────────────────────┐
│  👤 Alice                               │
│  ✅ 她擅长: 前端, React                  │
│  🔍 她在找: 后端, 数据库                  │
│  💡 互补度: 85%                          │
│  🏷️ 共同领域: AI, 教育                   │
│  💬 "想做一个AI辅助学习的项目"             │
│  [查看详情] [标记兴趣] [邀请组队]          │
└──────────────────────────────────────────┘
```

## 七、组队流程设计

### 7.1 两种组队路径

**路径 A：队伍优先（有想法的人建队）**
```
1. 创建队伍 → 填写项目想法 + 技能需求槽位
2. 队伍展示在"招募中"列表 + 图谱中
3. 参与者浏览队伍 → 申请加入
4. 队长审核 → 接受/拒绝
5. 满员或手动锁定
```

**路径 B：人员优先（先找人后建队）**
```
1. 浏览图谱 → 标记多个感兴趣的人
2. 查看谁也对自己感兴趣（双向兴趣）
3. 与感兴趣的人发起组队邀请
4. 多人确认后自动创建队伍
5. 选定队长 + 确定方向
```

### 7.2 队伍状态机

```
       创建
        │
        ▼
  ┌─ recruiting ─┐
  │   招募中      │◀─── 有人退出
  └──────┬───────┘
         │ 满员/手动
         ▼
  ┌─── full ────┐
  │   满员待确认  │
  └──────┬───────┘
         │ 全员确认
         ▼
  ┌── locked ───┐
  │   已锁定     │──── 活动开始
  └─────────────┘
```

### 7.3 约束规则

| 规则 | 默认值 | 可配置 |
|------|--------|--------|
| 每人同时只能属于 1 支队伍 | ✓ | ✓ |
| 队伍最小人数 | 2 | ✓ |
| 队伍最大人数 | 5 | ✓ |
| 组队截止时间 | 活动设定 | ✓ |
| 未组队参与者处理 | 进入"自由人池"供匹配 | ✓ |

## 八、可视化设计

### 8.1 全局图谱（继承 ClawMatch D3 力导向图）

在 ClawMatch 基础上增强：

| 节点类型 | 颜色 | 大小含义 | 新增信息 |
|----------|------|----------|----------|
| **人员** | 绿色 | 固定 | 标注 offer/want 技能、组队状态 |
| **技能标签** | 紫色 | ∝ 关联人数 | 区分 offer/want 边的粗细 |
| **队伍** | 橙色 | ∝ 队伍人数 | 显示招募槽位 |

**边类型**：
- 人 → 标签（offer）：实线
- 人 → 标签（want）：虚线
- 标签 ↔ 标签（关联）：浅色线
- 人 → 队伍（成员）：粗线
- 人 ─ ─ 人（双向兴趣）：高亮虚线

### 8.2 个人视角（以"我"为中心的子图）

```
          [React]──[前端]──[Alice]
            │                  │
           offer             want
            │                  │
          [我] ─── want ──── [后端]──[Bob]
            │                         │
           offer                    offer
            │                         │
         [Python]──[数据]──[Carol]
```

突出显示与"我"互补度最高的人，帮助用户快速定位潜在队友。

### 8.3 队伍总览视图

```
┌─ Team Alpha ──────────┐  ┌─ Team Beta ──────────┐
│ 🎯 AI教育助手          │  │ 🎯 开源社区管理        │
│ 👥 3/5 人              │  │ 👥 4/4 人 ✅ 满员     │
│ ├─ Alice (前端)        │  │ ├─ Eve (后端)         │
│ ├─ Bob (后端)          │  │ ├─ Frank (前端)       │
│ └─ Carol (设计)        │  │ ├─ Grace (设计)       │
│ 🔍 还缺: 数据, PM      │  │ └─ Henry (PM)        │
│ [申请加入]              │  │ [已满员]              │
└────────────────────────┘  └───────────────────────┘
```

| 前端 | Vite + Vanilla JS (或 Vue/React) | 继承 ClawMatch 技术栈，低门槛 |
| 可视化 | D3.js | 继承 ClawMatch，成熟的力导向图方案 |

## 十二、与 ClawMatch 的关键差异总结

| 维度 | ClawMatch | If-you-are-the-one |
|------|-----------|---------------------|
| **定位** | 通用社交图谱 | 活动组队专用 |
| **核心动作** | 展示、浏览 | 匹配、组队 |
| **Profile** | 单一技能标签列表 | offer/want 双向技能 + 风格/角色标签 |
| **匹配** | 被动（看图自己发现） | 主动（推荐引擎 + Like 机制） |
| **Team** | 无 | 一等公民实体 |
| **Canvas** | 长期多工作空间 | 短期单活动空间（生命周期） |
| **图谱** | 人-标签二部图 | 人-标签-队伍三层图 |
| **生命周期** | 持久 | 活动阶段流转 |
| **入口** | 注册登录 | 邀请码 → 快速档案 |
